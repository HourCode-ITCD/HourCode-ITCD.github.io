All about Floats

Float is a CSS positioning attribute. Print design can help us comprehend its purpose and origin. Images in a print layout can be placed into the page so that text wraps around them as needed. This is widely and aptly referred to as "text wrap." Here's an example:

The boxes that carry the text in page layout applications can be set to honor or ignore the text wrap. Ignoring the text wrap allows the words to flow over the image as if it didn't exist. This is the distinction between that picture being part of the page's flow and not being part of it (or not). The process of creating a website is pretty similar.

Page components with the CSS float attribute applied to them in web design behave similarly to pictures in print layouts, where the text flows around them. Floated items stay part of the web page's flow. This is not the same as page elements that employ absolute positioning. Page components that are absolutely positioned are detached from the flow of the webpage, such as when the text box in the print layout is directed to ignore the page wrap. Page elements that are absolutely positioned have no effect on the position of other components, and other elements have no effect on them, whether they touch or not.

What are floats used for?

Aside from the simple example of wrapping text around images, floats can be used to create entire web layouts.

Floats are also useful for designing in smaller situations. Consider this small section of a web page. If we use float for our small avatar picture, the text in the box will reflow to compensate as the image changes size:

Clearing the Float

Float's sister property is clear. An element that has the clear property set on it will not move up adjacent to the float like the float desires, but will move itself down past the float. Again an illustration probably does more good than words do.

The sidebar in the preceding example is floating to the right and is shorter than the main text area. The footer must then jump up into the open area as directed by the float. To resolve this issue, the footer may be cleared such that it remains beneath both floating columns.

Clear also has four permissible values. Both are the most often utilized methods for clearing floats coming from either direction. Left and Right may only be used to clear the float in one direction. None is the default, which is usually unnecessary unless a clear value is being removed from a cascade. Inherit is the fifth option, however it is oddly not supported by Internet Explorer. While clearing simply the left or right float is less prevalent in the wild, it has its purposes.

Techniques for Clearing Floats

If you always know what the subsequent element will be, you can apply the clear: both; value to that element and go on with your business. This is good since it does not require any fancy hacks or additional pieces, making it entirely semantic. Of course, that isn't always the case, and we need more float-clearing techniques in our repertoire.

The Overflow Method relies on setting the overflow CSS property on a parent element. If this property is set to auto or hidden on the parent element, the parent will expand to contain the floats, effectively clearing it for succeeding elements. This method can be beautifully semantic as it may not require additional elements. However if you find yourself adding a new div just to apply this, it is equally as non-semantic as the empty div method and less adaptable. Also bear in mind that the overflow property isn’t specifically for clearing floats. Be careful not to hide content or trigger unwanted scrollbars.

Positioning Types: How Do They Differ?

Static

This is the default for all page elements. Distinct elements do not have different positioning default values; they all start off static. The term "static" doesn't mean anything; it just signifies that the element will appear on the page as it usually would. The only time you'd set an element to position: static; is to aggressively remove some positioning that has been applied to an element outside of your control. This is uncommon since placement does not cascade.

Relative

This is most likely the most perplexing and misunderstood sort of posture. It truly means "related to itself." If you set position: relative; on an element but no other positioning attributes (top, left, bottom, or right), it will behave exactly as if you left it as position: static; However, if you do give it some other positioning attribute, say, top: 10px;, it will shift its position 10 pixels down from where it would normally be. As you may guess, the ability to move an element depending on its usual location is rather helpful.

When you set position, two things happen: relative; on an element that you should be aware of. One advantage is that it allows you to utilize z-index on that element, which isn't possible with statically positioned elements. Even if no z-index value is specified, this element will now appear on top of any other statically positioned element. You can't resist it by increasing the z-index of a statically positioned element.

Absolute

This is an extremely powerful sort of placement that allows you to virtually place any page element anywhere you want it. To set the location, you utilize the positioning attributes top, left, bottom, and right. Remember that with relative (or absolute) placement, these values will be relative to the next parent element. If no such parent exists, it will default all the way up to the html> element itself, implying that it will be inserted relative to the page itself.

The disadvantage of absolute placement (and the most crucial thing to remember) is that these items are disconnected from the flow of elements on the page. An element with this style of placement is unaffected by other elements and has no effect on them. This is something to think about every time you employ absolute positioning. Its overuse or wrong usage might restrict your site's versatility.

Fixed

A fixed position element is one that is positioned in relation to the viewport, or the browser window itself. Because the viewport does not change when the window is scrolled, a fixed positioned element will remain in place when the page is scrolled.

This may be used for things like a navigation bar that has to be displayed at all times regardless of where the page is scrolled. The issue with fixed placement is that it might lead to circumstances in which the fixed element overlays content, rendering it inaccessible.

Sticky

Sticky positioning is truly one-of-a-kind! A sticky element will sit like a static element, but when you scroll past it, if its parent element has room (usually: additional height), the sticky element will act as if it's fixed until that parent element runs out of room. In terms like that, it seems strange, but it's simple to see what's going on in a demo.

CSS Box Model

The CSS box model is a container that includes several attributes such as borders, margins, padding, and the content itself. It is used to develop web page design and layout. It may be used as a toolkit for modifying the arrangement of various parts. According to the CSS box model, the web browser presents each element as a rectangular box. In CSS, the Box-Model has several characteristics. Some of them are as follows:

content: This property is used to displays the text, images, etc, that can be sized using the width & height property.

padding: This property is used to create space around the element, inside any defined border.

border: This property is used to cover the content & any padding, & also allows to set the style, color, and width of the border.

margin: This property is used to create space around the element ie., around the border area.

The following figure illustrates the Box model in CSS.

Content Area: This area consists of content like text, images, or other media content. It is bounded by the content edge and its dimensions are given by content-box width and height.

Padding Area: It includes the element's padding. This area is actually the space around the content area and within the border-box. Its dimensions are given by the width of the padding-box and the height of the padding-box.

Border Area: It is the area between the box's padding and margin. Its dimensions are given by the width and height of the border.

Margin Area: This area consists of space between border and margin. The dimensions of the Margin area are the margin-box width and the margin-box height. It is useful to separate the element from its neighbors.
