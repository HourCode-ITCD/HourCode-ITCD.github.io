Radix UI

Radix UI is an open-source UI component toolkit for creating high-quality, accessible online apps and design systems. Radix Primitives is a low-level user interface component library that emphasizes accessibility, customisation, and developer experience. These components can be used as the foundation of your design system or added to it gradually.

Key features

Accessible

When feasible, components follow the WAI-ARIA design patterns. Many of the tough accessibility implementation elements, such as aria and role characteristics, attention management, and keyboard navigation, are handled by us. See our accessibility overview for more information.

Unstyled

Components come with no styles, providing you total stylistic control. CSS-in-JS libraries, CSS preprocessors, and vanilla CSS are all compatible with components. More information may be found in our style guide.

Opened

Radix Primitives are made to be tailored to your specifications. Our open component design gives you granular access to any component item, allowing you to wrap them and customize them with your own event listeners, properties, and references.

Uncontrolled

Components are unregulated by default where applicable, but can be controlled if desired. All of the behavior wiring is handled internally, allowing you to get up and going quickly without having to establish any local states.

Developer experience

One of our key objectives is to deliver the greatest development experience possible. The Radix Primitives API is completely typed. Because all of the components use the same API, the user experience is consistent and predictable. We've also added the asChild property, which gives users complete control over the displayed element.

Incremental adoption

Each primitive can be implemented separately, allowing for gradual adoption. Primitives are also versioned separately to make gradual adoption easier.

Animating with CSS animation

CSS is the simplest technique to animate Primitives. Both the mount and unmount processes may be animated with CSS animation. Because the Radix Primitives will postpone unmounting until your animation plays out, the latter is achievable.


Accessibility

Radix Primitives adhere to the WAI-ARIA writing requirements and have been tested in a variety of current browsers and assistive technologies.

WAI-ARIA

The W3C's WAI-ARIA specification outlines the semantics for several popular UI patterns that appear in Radix Primitives. This is intended to give meaning to controls that aren't developed with browser-provided components. If you build a button with a div rather than a button element, you'll need to add attributes to the div to indicate that it's a button for screen readers or speech recognition software.


Accessible Labels

The native HTML label element is meant to give semantic meaning and context for matching input components with several built-in form features. WAI-ARIA offers a protocol for providing accessible names and descriptions to non-form control elements, as well as custom controls like those given by Radix Primitives.
