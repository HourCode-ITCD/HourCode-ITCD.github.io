Hypertext Transfer Protocol Secure (HTTPS)

HTTPS (hypertext transfer protocol secure) is a secure method of transmitting data between a web server and a browser. It's a secure version of HTTP, the most used protocol for sending data between a web browser and a website. To strengthen the security of data transport, HTTPS is encrypted. This is especially crucial when users send sensitive information over the internet, such as when login into a bank account, email service, or health insurance provider.

HTTPS should be used on any website, especially those that need login credentials. Websites that do not employ HTTPS are indicated differently in current web browsers like Chrome than those that do. A green padlock in the URL bar indicates that the webpage is safe. HTTPS is taken seriously by web browsers; Google Chrome and other browsers mark all non-HTTPS websites as insecure.

How does HTTPS work?

To encrypt communications, HTTPS employs an encryption protocol. TLS stands for Transport Layer Security, however it was previously known as Secure Sockets Layer (SSL). An asymmetric public key infrastructure is used to secure communications in this protocol. This sort of security mechanism encrypts communications between two parties using two separate keys:

1. Private Key - this key is held by the website owner and kept confidential, as the reader may have guessed. This key is stored on a web server and is used to decode data that has been encrypted using the public key.

2. Public Key - Anyone who wishes to interact with the server in a secure manner can use this key. Only the private key may decode information encrypted with the public key.

Why is HTTPS important?

HTTPS protects websites from having their data broadcast in a way that anybody spying on the network can see. When data is transferred via standard HTTP, it is split down into packets of data that can be readily "sniffed" with free software. Communication through an insecure channel, such as public Wi-Fi, is therefore extremely vulnerable to interception. In reality, all HTTP communications are in plain text, making them extremely accessible to anybody with the right tools and open to on-path assaults.

  • HTTPS protects the integrity of your website
  • HTTPS protects the privacy and security of your users
  • HTTPS is the future of the web
