Cross-Origin Resource Sharing (CORS)

CORS is an HTTP header-based method that allows a server to specify any origins (domain, scheme, or port) other than its own from which a browser should allow resources to be loaded. CORS is an HTTP header-based system that allows a server to declare any origins other than its own from which a browser should allow resources to be loaded. CORS also uses a system in which browsers send a "preflight" request to the server hosting the cross-origin resource to ensure that it will allow the real request. The browser transmits headers indicating the HTTP method and headers that will be used in the real request during this preflight.

The CORS mechanism allows browsers and servers to make safe cross-origin requests and data transfers. To limit the hazards of cross-origin HTTP requests, modern browsers employ CORS in APIs like XMLHttpRequest and Fetch.

Funtional Overview

The Cross-Origin Resource Sharing standard works by allowing servers to specify which sources are allowed to access information from a web browser using new HTTP headers. Furthermore, for HTTP request methods that can have side effects on server data, such as HTTP methods other than GET or POST with certain MIME types, the specification requires that browsers "preflight" the request by using the HTTP OPTIONS request method to solicit supported methods from the server, and then sending the actual request after receiving "approval" from the server. Clients can additionally be informed if "credentials" like Cookies and HTTP Authentication should be transmitted with requests by servers.
