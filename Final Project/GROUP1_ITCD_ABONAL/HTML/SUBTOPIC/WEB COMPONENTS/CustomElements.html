CUSTOM ELEMENTS

What's The Point Of Custom Elements?

The basic idea is that if you design an element that constantly performs the same duty and has the same set of attributes and functions applied to it, you should be able to name it after the function it does. We have the video element, which displays video, the select element, which displays a select box, and the img element, which displays photos (and saving us from typing two characters whenever we write it). Many components define their own function.

How Do Custom Elements Work With Other Web Components Features?

Custom Elements have been developed to work in tandem with the Web Components suite's associated capabilities (and other generally related features). You might, for example, insert markup in the template element, which would not be processed by the browser until the element was activated.

With Shadow DOM, you can ensure that your code is isolated from the browser and hidden from the end user, and utilizing HTML Imports to share your element across many files and websites is a piece of cake.

Can We Use Custom Elements Today?

Well, no. And yes. These aren't just pipe dreams; browser vendors are actually working on them: the registerElement() function has been introduced in the current editions of Chrome and Opera, and it has also recently debuted in Firefox Nightly. However, raw Custom Elements are not yet suitable for usage in production.

However, there is a way around this, and that's to use Polymer. In case you haven't heard of it, it's an open community project set up to make future Web technologies usable today, and that includes Web Components and, through them, Custom Elements. Polymer is both a development library, which uses native implementations where available and polyfills where not, and a UI library, with common elements and patterns built using its own technology.
