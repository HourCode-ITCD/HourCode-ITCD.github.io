Module Bundlers

Module bundler is required because:

  • Browser does not support module system, although this is not entirely true nowadays
  • It helps you manage the dependency relationship of your code, it will load modules in dependency order for you.
  • It helps you to load your assets in dependency order, image asset, css asset, etc.

Module bundlers are the way to organize and combine many files of JavaScript code into one file. A JavaScript bundler can be used when your project becomes too large for a single file or when you're working with libraries that have multiple dependencies

esbuild

It is an extremely fast JavaScript and CSS bundler and minifier. Current build tools for the web are 10-100x slower than they could be. The main goal of this project is to bring about a new era of build tool performance, and create an easy-to-use modern bundler along the way. Esbuild is a tool in the JS Build Tools / JS Task Runners category of a tech stack. Esbuild is an open source tool with 31.4K GitHub stars and 806 GitHub forks.

esbuild is an extremely fast JavaScript/CSS bundler and minifier. It was created in June 2016, and has 30k stars on GitHub and 2.1 million weekly downloads from the npm registry.

Webpack

Webpack is a module bundler. Its main purpose is to bundle JavaScript files for usage in a browser, yet it is also capable of transforming, bundling, or packaging just about any resource or asset.

Webpack goes through your package and creates what it calls a dependency graph which consists of various modules which your webapp would require to function as expected. Then, depending on this graph, it creates a new package which consists of the very bare minimum number of files required, often just a single bundle.js file which can be plugged in to the html file easily and used for the application.
